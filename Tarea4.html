// 01 · Fundamentos

¿Qué es CSS3?

CSS (Cascading Style Sheets) es el lenguaje que controla la presentación visual de los documentos HTML. Si HTML es el esqueleto de una página web, CSS es la piel, la ropa y el maquillaje — define colores, tipografías, espaciados, tamaños, posiciones y animaciones de cada elemento.

CSS3 es la versión moderna del estándar, dividida en módulos independientes que se desarrollan y actualizan por separado. Introdujo capacidades transformadoras: gradientes, sombras, animaciones, transiciones, transformaciones 3D, Flexbox, Grid Layout, variables nativas y consultas de medios avanzadas. Todo sin necesidad de JavaScript ni plugins externos.

CSS funciona mediante un sistema de reglas: cada regla tiene un selector (qué elemento afecta) y una declaración (qué propiedad cambia y a qué valor). El navegador lee todas las reglas aplicables a cada elemento y decide cuál prevalece siguiendo el algoritmo de la cascada.

¿Cómo se vincula CSS a HTML? Hay tres formas: (1) Archivo externo <link rel="stylesheet" href="styles.css"> — la más recomendada. (2) Bloque interno <style> en el head. (3) Inline con el atributo style="" — la menos recomendada para proyectos reales.

// 02 · Base

Sintaxis de CSS

Una regla CSS se compone de un selector y un bloque de declaraciones. Cada declaración contiene una propiedad y su valor, separados por dos puntos, y terminan en punto y coma.

sintaxis-basica.css
1/* Selector → bloque de declaraciones */


2h1 {


3  color: #e50914;          /* propiedad: valor; */


4  font-size: 2rem;


5  text-transform: uppercase;


6  letter-spacing: 0.1em;


7}


8


9/* Múltiples selectores — mismas reglas */


10h1, h2, h3 {


11  font-family: 'Bebas Neue', sans-serif;


12  margin-bottom: 1rem;


13}


14


15/* Propiedad shorthand — varias en una */


16.tarjeta {


17  margin: 10px 20px 10px 20px; /* top right bottom left */


18  border: 1px solid #e50914;


19  padding: 24px;


20}

// 03 · Selectores

Selectores CSS

Los selectores definen qué elementos HTML serán estilizados. Son la parte más importante de CSS — dominarlos permite escribir código limpio, reutilizable y específico sin conflictos.

5

Categorías de Selectores

Básicos · Combinadores · Atributo · Pseudo-clases · Pseudo-elementos

* Universal

Selecciona todos los elementos del documento. Usado para resets y estilos base globales. Cuidado con el rendimiento en documentos grandes.

* { box-sizing: border-box; margin: 0; }

elemento Tipo

Selecciona todos los elementos de ese tipo HTML. p selecciona todos los párrafos, h1 todos los títulos H1.

p { color: #eee; line-height: 1.7; }

.clase Clase

Selecciona elementos con ese atributo class. El selector más usado en CSS. Un elemento puede tener múltiples clases. Reusable en toda la página.

.btn { padding: 10px 24px; cursor: pointer; }

#id ID

Selecciona el elemento con ese ID único. Alta especificidad — dificulta sobrescribir. Preferir clases para estilos reutilizables.

#hero { height: 100vh; overflow: hidden; }

A B Descendiente

Selecciona B que sea descendiente de A a cualquier nivel. Espacio entre selectores.

nav a { text-decoration: none; color: #e50914; }

A > B Hijo directo

Selecciona B que sea hijo directo (inmediato) de A. Más específico que el descendiente.

ul > li { list-style: none; padding: 8px 0; }

A + B Hermano adyacente

Selecciona B que inmediatamente sigue a A en el mismo nivel del DOM.

h2 + p { margin-top: 0; font-size: 1.1rem; }

A ~ B Hermano general

Selecciona todos los elementos B que siguen a A en el mismo nivel del DOM (no solo el inmediato).

h2 ~ p { color: rgba(255,255,255,0.6); }

[attr] Atributo

Selecciona elementos que tienen ese atributo. Variantes: [attr=valor] exacto, [attr^=val] comienza con, [attr$=val] termina con, [attr*=val] contiene.

input[type="email"] { border-color: #e50914; }

:hover Pseudo-clase

Se aplica cuando el cursor está sobre el elemento. Esencial para interactividad visual sin JavaScript.

.btn:hover { background: #e50914; color: #fff; }

:focus Pseudo-clase

Se aplica cuando el elemento tiene el foco (inputs, botones, links con Tab). Crítico para accesibilidad con teclado.

input:focus { outline: 2px solid #e50914; }

:nth-child(n) Pseudo-clase

Selecciona el enésimo hijo. :nth-child(odd) impares, :nth-child(even) pares, :nth-child(3n) cada tercero.

tr:nth-child(even) { background: rgba(255,255,255,0.03); }

:not(selector) Pseudo-clase

Selecciona todos los elementos que NO coinciden con el selector dado. Muy útil para excluir casos específicos.

.nav-link:not(.active) { opacity: 0.5; }

::before / ::after Pseudo-elemento

Inserta contenido virtual antes o después del elemento. Requieren content. Usados para decoraciones, badges e iconos sin HTML extra.

.btn::after { content: '→'; margin-left: 8px; }

::placeholder Pseudo-elemento

Estiliza el texto placeholder de inputs y textareas. Permite personalizar su color, fuente y opacidad.

input::placeholder { color: rgba(255,255,255,0.3); }

// 04 · La Cascada

Cascada y Especificidad

La cascada es el algoritmo que CSS usa para decidir qué regla prevalece cuando múltiples reglas se aplican al mismo elemento. Tiene tres factores en orden de prioridad: importancia (!important), especificidad (peso del selector) y orden de aparición (la última gana entre iguales).

SelectorEspecificidadValor (a,b,c)
*Ninguna0,0,0
elemento (h1, p, div)Muy baja0,0,1
.clase, :pseudo-clase, [attr]Baja0,1,0
#idAlta1,0,0
style="" (inline)Muy alta1,0,0,0
!importantMáxima — rompe la cascadaAnula todo
⚠

Evita !important salvo en casos excepcionales. Su uso excesivo hace el CSS imposible de mantener. Si necesitas !important frecuentemente, es señal de que la arquitectura del CSS tiene problemas de especificidad.

especificidad.css
1/* Especificidad: 0,0,1 — tipo */


2p { color: white; }


3


4/* Especificidad: 0,1,0 — clase (gana) */


5.texto { color: red; }


6


7/* Especificidad: 0,1,1 — clase + tipo (gana) */


8p.texto { color: #e50914; }


9


10/* Herencia: color y font se heredan */


11body {


12  color: #eeeeee;       /* todos los hijos heredan */


13  font-family: sans-serif;


14}

// 05 · Box Model

El Modelo de Caja

Todo elemento HTML es una caja rectangular. El Box Model describe las capas que componen esa caja de adentro hacia afuera: content (el contenido), padding (relleno interior), border (borde) y margin (espacio exterior). Entender esto es fundamental para dominar el layout en CSS.

box-model.css
1/* box-sizing: border-box — el más usado */


2* { box-sizing: border-box; }


3/* Con border-box, width/height incluyen padding y border */


4


5.caja {


6  width:   300px;       /* ancho del contenido */


7  height:  200px;


8  padding: 24px;       /* relleno interior (4 lados) */


9  border:  2px solid #e50914;


10  margin:  16px auto;  /* centrado horizontal */


11}


12


13/* Shorthand padding: top right bottom left */


14.asimetrico {


15  padding: 10px 20px;   /* top/bottom=10, left/right=20 */


16  margin-top: 0;         /* propiedad individual */


17  margin-inline: auto;  /* CSS moderno: centrar */


18}

width / height

Dimensiones del contenido. Valores: px, rem, em, %, vw/vh. min-width, max-width, min-height, max-height para límites.

width: 100%; max-width: 1200px;

padding

Espacio interior entre el contenido y el borde. Afecta el tamaño visual pero no separa del resto. Shorthand: 1 valor (todos), 2 (v/h), 4 (top right bottom left).

padding: 16px 24px;

margin

Espacio exterior entre el elemento y sus vecinos. margin: auto centra elementos de bloque. Los márgenes verticales colapsan entre sí.

margin: 0 auto; /* centrado */

overflow

Controla qué sucede cuando el contenido supera las dimensiones de la caja. Valores: visible, hidden, scroll, auto.

overflow: hidden; overflow-x: auto;

// 06 · Display & Posición

Display y Posicionamiento

display: block

Ocupa todo el ancho disponible, empieza en nueva línea. Ej: div, p, h1-h6. Acepta width/height.

display: inline

Solo ocupa el espacio de su contenido, fluye con el texto. No acepta width/height. Ej: span, a, strong.

display: inline-block

Fluye como inline pero acepta width/height y padding vertical. Útil para botones y badges.

display: inline-block; padding: 8px 16px;

display: none

Oculta el elemento completamente — no ocupa espacio en el layout. Diferente a visibility: hidden que oculta pero conserva el espacio.

position: static

Valor por defecto. El elemento fluye normalmente. Las propiedades top/left/right/bottom no tienen efecto.

position: relative

Se posiciona relativo a su posición normal. Conserva su espacio en el flujo. Crea contexto para absolute hijos.

position: relative; top: 10px; left: 5px;

position: absolute

Se posiciona relativo al ancestro positioned más cercano. Sale del flujo normal — no ocupa espacio. Ideal para overlays y badges.

position: absolute; top: 0; right: 0;

position: fixed

Se posiciona relativo al viewport. No se mueve al hacer scroll. Para navbars fijos, botones flotantes, modals.

position: fixed; bottom: 20px; right: 20px;

position: sticky

Alterna entre relative y fixed según el scroll. Se pega al viewport cuando llega al umbral definido. Para headers y sidebars.

position: sticky; top: 0; z-index: 100;

z-index

Controla el orden de apilamiento en el eje Z. Solo funciona en elementos con position distinto de static. Mayor número = encima.

z-index: 9999; /* modal encima de todo */

// 07 · Flexbox

Flexbox — Diseño Flexible

Flexbox (Flexible Box Layout) es un modelo de layout unidimensional — trabaja en una sola dirección a la vez (fila o columna). Es la herramienta ideal para alinear elementos, distribuir espacio y crear layouts de componentes: navbars, cards, botones, centrado perfecto.

flexbox.css
1/* Contenedor flex */


2.navbar {


3  display: flex;


4  flex-direction: row;         /* row | column */


5  justify-content: space-between; /* eje principal */


6  align-items: center;         /* eje cruzado */


7  gap: 16px;                   /* espacio entre ítems */


8  flex-wrap: wrap;             /* rompe a nueva línea */


9}


10


11/* Ítem flex */


12.nav-logo {


13  flex: none;  /* no crece ni se reduce */


14}


15.nav-links {


16  flex: 1;      /* ocupa el espacio disponible */


17}


18


19/* Centrado perfecto con flex */


20.centrado {


21  display: flex;


22  align-items: center;


23  justify-content: center;


24}

justify-content

Alinea ítems en el eje principal (horizontal en row). Valores: flex-start, flex-end, center, space-between, space-around, space-evenly.

align-items

Alinea ítems en el eje cruzado (vertical en row). Valores: flex-start, flex-end, center, stretch, baseline.

flex-direction

Define la dirección del eje principal: row (izq→der), row-reverse, column (arriba→abajo), column-reverse.

flex-wrap

Controla si los ítems se rompen a una nueva línea cuando no caben: nowrap (default), wrap, wrap-reverse.

flex: 1

Shorthand de flex-grow, flex-shrink y flex-basis. flex: 1 hace que el ítem crezca para llenar el espacio disponible.

align-self

Anula align-items para un ítem específico. Permite excepciones dentro del contenedor flex.

align-self: flex-end; /* solo este ítem */

order

Cambia el orden visual de los ítems sin modificar el HTML. Defecto: 0. Negativo para mover antes, positivo para mover después.

order: -1; /* primero visualmente */

gap

Espacio entre ítems flex (y grid). Reemplaza los márgenes entre ítems. Acepta dos valores: gap: row-gap column-gap.

gap: 20px; gap: 16px 24px;

// 08 · CSS Grid

CSS Grid — Layout Bidimensional

CSS Grid es el sistema de layout más poderoso de CSS. A diferencia de Flexbox (unidimensional), Grid trabaja en dos dimensiones simultáneamente — filas y columnas. Es ideal para layouts de página completa, galerías y cualquier estructura compleja.

css-grid.css
1/* Grid básico de 3 columnas */


2.galeria {


3  display: grid;


4  grid-template-columns: 1fr 1fr 1fr;  /* 3 col iguales */


5  grid-template-rows: auto;


6  gap: 20px;


7}


8


9/* Grid responsivo con auto-fill */


10.cards {


11  display: grid;


12  grid-template-columns:


13    repeat(auto-fill, minmax(280px, 1fr));


14  gap: 24px;


15}


16


17/* Ítem que ocupa varias columnas */


18.destacado {


19  grid-column: 1 / 3;  /* de col 1 a col 3 */


20  grid-row:    1 / 2;


21}


22


23/* Layout de página con grid-template-areas */


24.pagina {


25  display: grid;


26  grid-template-areas:


27    "header  header"


28    "sidebar main"


29    "footer  footer";


30  grid-template-columns: 220px 1fr;


31}


32.header  { grid-area: header; }


33.sidebar { grid-area: sidebar; }

// 09 · Tipografía

Propiedades Tipográficas

font-family

Define la fuente. Lista de fuentes de respaldo separadas por comas. Siempre terminar con genérica: serif, sans-serif, monospace.

font-family: 'Bebas Neue', sans-serif;

font-size

Tamaño de fuente. Unidades recomendadas: rem (relativo a raíz), em (relativo al padre), clamp() para fluido.

font-size: clamp(1rem, 2.5vw, 1.5rem);

font-weight

Grosor de la fuente. Valores numéricos 100-900 (si la fuente los soporta) o keywords: normal(400), bold(700).

font-weight: 300; /* light */

line-height

Altura de línea. Sin unidades es relativo al font-size del elemento (recomendado). 1.5–1.7 para texto de lectura.

line-height: 1.7;

letter-spacing

Espacio entre caracteres. Positivo aumenta, negativo reduce. em es la unidad más consistente para este propósito.

letter-spacing: 0.05em;

text-align

Alineación horizontal del texto: left, right, center, justify. Solo afecta el contenido inline dentro del bloque.

text-transform

Transforma el texto: uppercase, lowercase, capitalize. Semánticamente mejor que escribir en mayúsculas en HTML.

text-transform: uppercase;

text-decoration

Subrayado, tachado, sobrerayado. none elimina el subrayado de los links. También controla color y estilo de la línea.

text-decoration: none; /* links sin subrayado */

white-space

Controla cómo se manejan espacios y saltos de línea. nowrap evita que el texto se rompa, pre preserva el formato original.

white-space: nowrap; /* una línea */

text-overflow

Controla qué se muestra cuando el texto desborda su contenedor. Requiere overflow: hidden y white-space: nowrap.

text-overflow: ellipsis; /* texto... */

// 10 · Colores y Fondos

Colores, Gradientes y Fondos

colores.css
1/* Formatos de color en CSS */


2.elemento {


3  color: red;                   /* nombre */


4  color: #e50914;               /* hexadecimal */


5  color: rgb(229, 9, 20);       /* rgb */


6  color: rgba(229, 9, 20, 0.5); /* rgba con opacidad */


7  color: hsl(354, 93%, 47%);   /* hsl (tono,sat,luz) */


8}


9


10/* Gradientes */


11.gradiente-lineal {


12  background: linear-gradient(


13    135deg, #060608 0%, #e50914 100%


14  );


15}


16.gradiente-radial {


17  background: radial-gradient(


18    ellipse at center,


19    rgba(229,9,20,.3) 0%, transparent 70%


20  );


21}

background-color

Color de fondo sólido del elemento. Acepta todos los formatos de color CSS. transparent para sin fondo.

background-image

Imagen o gradiente de fondo. Acepta url(), linear-gradient(), radial-gradient(). Se pueden combinar múltiples separando con comas.

background-size

Tamaño de la imagen de fondo. cover llena sin deformar, contain cabe completa, o valores explícitos.

background-size: cover;

background-position

Posición de la imagen de fondo: center, top left, coordenadas (50% 50%).

opacity

Transparencia de todo el elemento (0=invisible, 1=sólido). Afecta el elemento y todos sus hijos. Para solo el fondo, usar rgba.

opacity: 0.5;

backdrop-filter

Aplica efectos visuales al área detrás del elemento. blur(), brightness(), saturate(). Efecto "vidrio esmerilado".

backdrop-filter: blur(12px);

// 11 · Bordes y Sombras

Bordes, Radios y Sombras

border

Shorthand: border: grosor estilo color. Estilos: solid, dashed, dotted, double, none. Lados individuales con border-top, border-right, etc.

border: 1px solid #e50914;

border-radius

Esquinas redondeadas. 50% crea un círculo perfecto. Valores independientes por esquina: top-left, top-right, bottom-right, bottom-left.

border-radius: 8px; border-radius: 50%;

box-shadow

Sombra de caja. Sintaxis: x y blur spread color. inset para sombra interior. Múltiples sombras separadas por comas.

box-shadow: 0 0 20px rgba(229,9,20,0.4);

text-shadow

Sombra sobre texto. Sintaxis: x y blur color. Sin parámetro spread. Múltiples sombras dan efecto de glow.

text-shadow: 0 0 30px rgba(229,9,20,0.6);

outline

Similar a border pero fuera del box model — no afecta el layout. Usado por navegadores para indicar foco. No remover sin reemplazar por accesibilidad.

outline: 2px solid #e50914; outline-offset: 2px;

filter

Efectos gráficos: blur(), brightness(), contrast(), grayscale(), saturate(), drop-shadow(). Aplica a toda la caja incluyendo contenido.

filter: grayscale(100%) brightness(0.8);

// 12 · Transformaciones

Transform — Transformaciones 2D y 3D

La propiedad transform aplica transformaciones geométricas a los elementos sin afectar el flujo del documento. Las transformaciones son aceleradas por la GPU, por lo que son la opción de rendimiento para animaciones.

transform.css
1.tarjeta {


2  /* Múltiples transforms en una línea */


3  transform: translateY(-6px) scale(1.02);


4  transform-origin: center;


5}


6


7/* Funciones de transform */


8translate(50px, 100px)  /* mover */


9translateX(-50%)       /* mover en X */


10scale(1.5)             /* escalar */


11rotate(45deg)          /* rotar */


12skewX(15deg)          /* sesgar */


13rotateY(180deg)       /* 3D: voltear */


14perspective(600px)    /* profundidad 3D */

// 14 · Variables

Custom Properties (Variables CSS)

Las Custom Properties (variables nativas de CSS) permiten definir valores reutilizables que se pueden cambiar desde un solo lugar. Se declaran con --nombre-variable y se usan con var(--nombre-variable). Son parte del DOM, heredables, y pueden modificarse con JavaScript.

variables.css
1/* Definir variables en :root (global) */


2:root {


3  --color-principal: #e50914;


4  --color-fondo:    #060608;


5  --fuente-display: 'Bebas Neue', sans-serif;


6  --espaciado-base: 1rem;


7  --radio-borde:    4px;


8}


9


10/* Usar variables */


11.boton {


12  background: var(--color-principal);


13  font-family: var(--fuente-display);


14  padding: var(--espaciado-base) calc(var(--espaciado-base) * 2);


15}


16


17/* Tema oscuro con variables */


18@media (prefers-color-scheme: dark) {


19  :root { --color-fondo: #000; }


20}


21


22/* Modificar con JS */

24//   .setProperty('--color-principal', '#0ff');

// 15 · Responsive Design

Diseño Responsivo

El diseño responsivo adapta la interfaz a diferentes tamaños de pantalla. El enfoque moderno es mobile-first: diseñar primero para móvil y luego expandir con media queries para pantallas más grandes.

responsive.css
1/* Mobile-first: estilos base para móvil */


2.grid {


3  display: grid;


4  grid-template-columns: 1fr;  /* 1 columna en móvil */


5  gap: 16px;


6}


7


8/* Tablet: 768px+ */


9@media (min-width: 768px) {


10  .grid { grid-template-columns: 1fr 1fr; }


11}


12


13/* Desktop: 1200px+ */


14@media (min-width: 1200px) {


15  .grid { grid-template-columns: repeat(4, 1fr); }


16}


17


18/* Unidades fluidas — sin media queries */


19.titulo {


20  /* min | ideal | max */


21  font-size: clamp(1.5rem, 5vw, 4rem);


22  padding: clamp(16px, 5vw, 80px);


23}


24


25/* Otros media queries útiles */


26@media (prefers-reduced-motion: reduce) {


27  * { animation-duration: 0.01ms !important; }


28}


29@media (hover: none) {


30  .btn:hover { /* evita hover en táctiles */ }


31}
"En CSS moderno, clamp(), auto-fill y minmax() construyen layouts responsivos sin una sola media query."

Breakpoints recomendados (mobile-first): 480px móvil grande · 768px tablet · 1024px laptop · 1280px desktop · 1536px pantalla grande. No son reglas fijas — los breakpoints deben definirse donde el diseño se rompe, no por dispositivo.
