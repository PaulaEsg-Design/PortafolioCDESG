// 01 · Fundamentos

¿Qué es HTML5?

HTML (HyperText Markup Language) es el lenguaje estándar para crear páginas web. No es un lenguaje de programación — es un lenguaje de marcado que define la estructura y el significado del contenido web. Cada elemento HTML indica al navegador qué tipo de contenido está mostrando: un título, un párrafo, una imagen, un enlace.

HTML5 es la quinta versión del estándar, lanzada oficialmente en octubre de 2014 por el W3C. Introdujo soporte nativo para audio, video, canvas 2D/3D, almacenamiento local, geolocalización y decenas de nuevas etiquetas semánticas que mejoran la accesibilidad y el SEO.

¿Cómo funciona? El navegador (Chrome, Firefox, Safari) lee el código HTML y lo convierte en la página visual que ves. Este proceso se llama renderizado. El navegador construye un árbol llamado DOM (Document Object Model) que representa todos los elementos de la página.

// 03 · Metadatos

DOCTYPE y Etiquetas del Head

El <head> es la sección invisible del documento. Contiene metadatos que le dicen al navegador y a los motores de búsqueda información sobre la página: su título, codificación, estilos, scripts y más.

<!DOCTYPE html> Declaración

No es una etiqueta HTML sino una instrucción al navegador. Le indica que use el estándar HTML5 para interpretar el documento. Debe ser la primera línea de todo archivo HTML.

<!DOCTYPE html>

<html> Raíz

Elemento raíz de todo documento HTML. El atributo lang define el idioma del contenido — crucial para accesibilidad y SEO.

<html lang="es">

<head> Contenedor

Contiene todos los metadatos del documento. Su contenido no se muestra en la página pero es fundamental para el funcionamiento correcto.

<head>...</head>

<title> SEO

Define el título de la página que aparece en la pestaña del navegador y en los resultados de búsqueda de Google. Máximo recomendado: 60 caracteres.

<title>Mi Página</title>

<meta charset> Codificación

Define el conjunto de caracteres del documento. UTF-8 soporta todos los idiomas del mundo incluyendo español, emojis y caracteres especiales.

<meta charset="UTF-8"/>

<meta viewport> Responsive

Controla cómo se escala la página en dispositivos móviles. Sin esta etiqueta, los celulares mostrarían la página como si fuera una pantalla de escritorio reducida.

<meta name="viewport" content="width=device-width, initial-scale=1.0"/>

<link> Externo

Vincula recursos externos al documento. Principalmente usado para conectar hojas de estilo CSS, pero también para favicons, fuentes y manifests de PWA.

<link rel="stylesheet" href="styles.css"/>

<script> JavaScript

Incluye código JavaScript en la página. Se puede colocar en el head (con defer) o al final del body. El atributo src enlaza un archivo externo.

<script src="app.js" defer></script>

<style> CSS inline

Permite escribir CSS directamente dentro del HTML. Útil para estilos específicos de página, aunque se recomienda usar archivos CSS externos para proyectos grandes.

<style> h1 { color: red; } </style>

// 04 · Contenido

Etiquetas de Texto y Tipografía

Las etiquetas de texto son las más fundamentales de HTML. Definen la jerarquía y el significado del contenido escrito. La regla principal: usa la etiqueta correcta según el significado semántico, no por cómo se ve visualmente — el estilo visual lo controla CSS.

H1–H6

Encabezados — Jerarquía de Títulos

6 niveles · solo uno H1 por página

<h1> Único

Título principal de la página. Solo debe haber uno por documento. El más importante para SEO y accesibilidad. Google lo usa para entender el tema principal.

<h1>Título Principal</h1>

<h2> — <h6> Jerarquía

Subtítulos en orden descendente de importancia. h2 para secciones principales, h3 para subsecciones, etc. Nunca saltes niveles (ej: de h1 a h3).

<h2>Sección</h2>
<h3>Subsección</h3>

<p> Párrafo

Define un párrafo de texto. El navegador añade automáticamente espacio antes y después. Es el elemento de texto más usado en HTML.

<p>Este es un párrafo de texto.</p>

<strong> Semántico

Indica importancia fuerte. Se renderiza en negrita pero su función real es semántica — los lectores de pantalla lo leen con énfasis. Prefiere <strong> sobre <b>.

<strong>Texto importante</strong>

<em> Énfasis

Énfasis semántico. Se muestra en cursiva. Indica que esa palabra es importante en el contexto de la oración. Usa <em> en lugar de <i> para énfasis real.

<em>Texto enfatizado</em>

<span> Inline

Contenedor genérico inline sin semántica. Se usa para aplicar estilos CSS o JavaScript a porciones de texto sin afectar el flujo del documento.

<span class="rojo">texto</span>

<div> Block

Contenedor genérico de bloque. Sin semántica propia. Es el más usado en HTML para crear layouts y agrupar elementos. Ocupa todo el ancho disponible.

<div class="contenedor">...</div>

<br> Salto

Insertar un salto de línea dentro de un texto. No separa párrafos — usa <p> para eso. Útil en poemas, direcciones o código donde el salto tiene significado.

Línea 1<br/>Línea 2

<hr> Separador

Línea horizontal de separación temática. Indica un cambio de tema en el contenido. Se puede estilizar con CSS para cambiar su apariencia.

<p>Tema A</p><hr/><p>Tema B</p>

<blockquote> Cita

Bloque de cita textual de otra fuente. El atributo cite indica la URL de origen. Útil para referencias académicas y testimonios.

<blockquote cite="url">Cita</blockquote>

<code> Código

Muestra fragmentos de código inline. El navegador usa una fuente monoespaciada. Para bloques de código multilinea usa <pre><code>.

<code>const x = 5;</code>

<pre> Preformateado

Preserva espacios en blanco y saltos de línea exactamente como están escritos. Ideal para bloques de código donde el formato visual importa.

<pre><code> x = 1</code></pre>

<mark> HTML5

Resalta texto marcado o relevante, como en resultados de búsqueda. Por defecto muestra fondo amarillo. Tiene significado semántico de "resultado relevante".

<mark>texto resaltado</mark>

<small> Legal

Texto de letra pequeña — comentarios legales, disclaimers, copyright. No solo es estético: tiene el significado semántico de "texto secundario".

<small>© 2026 Todos los derechos</small>

<del> / <ins> Edición

<del> tacha texto eliminado. <ins> subraya texto insertado. Usados en documentos con historial de cambios o versiones.

<del>texto viejo</del> <ins>nuevo</ins>

<abbr> Abreviatura

Define una abreviatura o acrónimo. El atributo title muestra el significado completo al pasar el cursor. Mejora accesibilidad y comprensión.

<abbr title="HyperText">HTML</abbr>

<time> HTML5

Define una fecha u hora legible por máquinas. El atributo datetime contiene el valor en formato ISO. Crucial para SEO de eventos.

<time datetime="2026-03-14">Hoy</time>

<sub> / <sup> Tipografía

<sub> subíndice (H₂O). <sup> superíndice (x²). Imprescindibles para fórmulas matemáticas y químicas.

H<sub>2</sub>O | x<sup>2</sup>

// 05 · Organización

Listas

<ul> No ordenada

Lista de puntos sin orden específico. Cada elemento se marca con <li>. El símbolo del punto (bullet) se controla con CSS.

<ul>
<li>HTML</li>
<li>CSS</li>
</ul>

<ol> Ordenada

Lista numerada. Los atributos start y type permiten controlar el número inicial y el tipo de numeración (1, A, a, I, i).

<ol start="3" type="A">
<li>Paso A</li>
</ol>

<li> Ítem

Elemento de lista. Debe ser hijo directo de <ul>, <ol> o <menu>. Puede contener cualquier tipo de contenido HTML.

<li>Texto del ítem</li>

<dl> / <dt> / <dd> Definición

Lista de definiciones. <dt> es el término, <dd> es la definición. Ideal para glosarios, diccionarios y listas clave-valor.

<dl>
<dt>HTML</dt>
<dd>Lenguaje de marcado</dd>
</dl>

// 06 · Media & Links

Enlaces e Imágenes

<a> Enlace

Crea un hipervínculo a otra página, sección o recurso. El atributo href define el destino. target="_blank" abre en nueva pestaña. rel="noopener" por seguridad.

<a href="https://..." target="_blank" rel="noopener">Visitar</a>

<img> Imagen

Inserta una imagen. src: ruta de la imagen. alt: texto alternativo (obligatorio para accesibilidad). width y height para dimensiones.

<img src="foto.jpg" alt="descripción" width="400"/>

<picture> HTML5

Contenedor para imágenes responsivas con múltiples fuentes. Permite servir diferentes imágenes según el tamaño de pantalla o soporte del navegador (WebP vs JPEG).

<picture>
<source srcset="img.webp" type="image/webp"/>
<img src="img.jpg" alt="foto"/>
</picture>

<figure> / <figcaption> Semántico

Contenedor para imágenes, diagramas o código con su leyenda. <figcaption> proporciona descripción accesible vinculada semánticamente a la figura.

<figure>
<img src="img.jpg" alt=""/>
<figcaption>Descripción</figcaption>
</figure>

<map> / <area> Mapa imagen

Crea mapas de imagen clicables donde diferentes zonas de una imagen son hipervínculos. <area> define cada zona clickeable.

<map name="mapa">
<area shape="rect" coords="0,0,100,100" href="#"/>
</map>

<iframe> Embebido

Inserta un documento HTML externo dentro de la página actual. Usado para embeber videos de YouTube, mapas de Google, widgets de redes sociales y más.

<iframe src="https://..." width="560" height="315" allowfullscreen></iframe>

Tipos de rutas en href/src: Las rutas absolutas incluyen el dominio completo (https://sitio.com/imagen.jpg). Las rutas relativas son relativas al archivo actual (../imagenes/foto.jpg). Las rutas raíz comienzan con / (/imagenes/foto.jpg).

// 07 · Datos

Tablas

Las tablas en HTML se usan para mostrar datos tabulares — información que tiene una relación entre filas y columnas. Nunca uses tablas para layout visual — eso se hace con CSS Grid o Flexbox.

<table>

Contenedor principal de la tabla. Todos los demás elementos de tabla deben estar dentro de él.

<thead>

Agrupa las filas de encabezado. Mejora la accesibilidad y permite al navegador repetir los encabezados al imprimir tablas largas.

<tbody>

Agrupa las filas de datos del cuerpo de la tabla. Si la tabla tiene thead/tfoot, debe usarse tbody para claridad semántica.

<tfoot>

Agrupa las filas del pie de la tabla. Útil para totales, sumatorias o notas al pie de los datos.

<tr>

Define una fila de la tabla. Contiene celdas <td> o <th>.

<th>

Celda de encabezado. Bold por defecto. El atributo scope="col" o scope="row" mejora la accesibilidad para lectores de pantalla.

<td>

Celda de datos. Los atributos colspan y rowspan permiten fusionar celdas horizontalmente o verticalmente.

<caption>

Título de la tabla. Debe ser el primer hijo de <table>. Mejora accesibilidad — los lectores de pantalla lo anuncian antes de leer la tabla.

// 08 · Interactividad

Formularios

Los formularios son el principal mecanismo de entrada de datos del usuario en la web. Con HTML5 se añadieron nuevos tipos de input que incluyen validación nativa, sin necesidad de JavaScript.

<form>

Contenedor del formulario. action define a dónde se envían los datos, method el método HTTP (GET o POST). novalidate desactiva validación nativa.

<form action="/enviar" method="POST">

<input> Versátil

El elemento de entrada más flexible. El atributo type define el comportamiento. Más de 20 tipos disponibles en HTML5. Auto-cierre (no necesita </input>).

<input type="text" name="usuario" placeholder="Tu nombre"/>

<label>

Etiqueta descriptiva para un campo. El atributo for debe coincidir con el id del input. Crucial para accesibilidad — permite hacer clic en la etiqueta para activar el campo.

<label for="email">Email:</label>
<input id="email" type="email"/>

<textarea>

Campo de texto multilínea. Los atributos rows y cols definen el tamaño inicial. El contenido predeterminado va entre las etiquetas de apertura y cierre.

<textarea rows="4" placeholder="Mensaje"></textarea>

<select> / <option>

Lista desplegable. <option> define cada opción. El atributo selected en una opción la marca como predeterminada. multiple permite selección múltiple.

<select name="pais">
<option value="mx" selected>México</option>
</select>

<button>

Botón interactivo. type="submit" envía el formulario, type="reset" lo limpia, type="button" para acciones JavaScript personalizadas.

<button type="submit">Enviar</button>

<fieldset> / <legend>

Agrupa campos relacionados con un borde y título. Mejora la organización visual y la accesibilidad de formularios largos y complejos.

<fieldset>
<legend>Datos personales</legend>
</fieldset>

<datalist> HTML5

Proporciona una lista de sugerencias para un campo input. Combina la libertad de un input de texto con las opciones predefinidas de un select.

<input list="lenguas"/>
<datalist id="lenguas">
<option value="HTML"/>
</datalist>

<output> HTML5

Muestra el resultado de un cálculo o acción. Usado con formularios interactivos donde el resultado cambia en tiempo real según las entradas del usuario.

<output name="resultado">0</output>

<progress> HTML5

Barra de progreso nativa. El atributo value indica el progreso actual y max el valor máximo. Completamente estilizable con CSS.

<progress value="70" max="100">70%</progress>

<meter> HTML5

Medidor escalar dentro de un rango conocido. Diferente a progress: representa un valor actual, no un progreso. Tiene atributos min, max, low, high.

<meter value="6" min="0" max="10">6/10</meter>

// 09 · Semántica

Etiquetas Semánticas HTML5

Las etiquetas semánticas de HTML5 reemplazaron el uso masivo de <div> con etiquetas que tienen significado real. Los buscadores, lectores de pantalla y otras herramientas entienden qué tipo de contenido contiene cada sección.

"Semántica HTML5: el arte de decirle al navegador qué significa tu contenido, no solo cómo se ve."

<header>

Encabezado de la página o de una sección. Contiene típicamente el logo, la navegación principal y el título. Puede haber múltiples header en una página (uno por section).

<nav>

Bloque de navegación principal. Contiene los enlaces de menú. Los lectores de pantalla usan esta etiqueta para ofrecer navegación rápida al usuario.

<main>

Contenido principal de la página — único por documento. Todo lo que no sea header, footer, nav, aside. Es el centro del documento para lectores de pantalla.

<section>

Sección temática del contenido con su propio encabezado. Más semántico que un div. Úsalo cuando el contenido tiene coherencia temática propia.

<article>

Contenido independiente y auto-contenido: un post de blog, una noticia, un comentario. Puede distribuirse o reutilizarse independientemente del contexto.

<aside>

Contenido secundario relacionado pero separable del principal: barras laterales, anuncios, notas al margen, biografía del autor. No es contenido principal.

<footer>

Pie del documento o de una sección. Contiene información de contacto, copyright, enlaces secundarios y mapa del sitio. Puede haber múltiples footers.

<details> / <summary>

Widget de divulgación nativo — acordeón sin JavaScript. <summary> es el título visible siempre. El contenido de <details> se muestra/oculta al clic.

<details>
<summary>Ver más</summary>
Contenido oculto aquí
</details>

<dialog> HTML5

Elemento de diálogo o modal nativo. El método JavaScript showModal() lo abre como modal. Incluye manejo de foco y tecla Escape automáticos.

<dialog id="mi-modal">
Contenido
</dialog>

<address>

Información de contacto del autor del documento o artículo. No solo para direcciones físicas — puede contener email, teléfono, redes sociales.

<address>[email protected]</address>

// 10 · Multimedia

Audio, Video y Canvas

<video> HTML5

Reproduce video nativo sin plugins. Atributos: controls muestra controles, autoplay auto-reproduce, loop repite, muted silencia, poster imagen de vista previa.

<video controls poster="preview.jpg">
<source src="video.mp4" type="video/mp4"/>
<source src="video.webm" type="video/webm"/>
</video>

<audio> HTML5

Reproduce audio nativo. Los mismos atributos que video excepto poster. Siempre incluir múltiples formatos (MP3 + OGG) para compatibilidad cruzada.

<audio controls>
<source src="audio.mp3" type="audio/mpeg"/>
<source src="audio.ogg" type="audio/ogg"/>
</audio>

<canvas> HTML5

Área de dibujo 2D/3D controlada por JavaScript. Usado para gráficos, juegos, visualizaciones y efectos visuales. La API Canvas 2D y WebGL se acceden desde JS.

<canvas id="mi-canvas" width="800" height="400"></canvas>

<svg>

Gráficos vectoriales escalables directamente en HTML. A diferencia del canvas, los elementos SVG son parte del DOM y se pueden estilar con CSS y manipular con JavaScript.

<svg width="100" height="100">
<circle cx="50" cy="50" r="40" fill="red"/>
</svg>

<source>

Define fuentes alternativas para video, audio o picture. El navegador elige la primera que soporta. Usa el atributo media para condiciones responsivas.

<source src="video.mp4" type="video/mp4"/>

<track>

Subtítulos y metadatos de tiempo para video/audio. El atributo kind define el tipo: subtitles, captions, descriptions, chapters, metadata.

<track src="subs.vtt" kind="subtitles" srclang="es" label="Español"/>

<embed> / <object>

Embeben contenido externo o plugins. <embed> es auto-cierre y más simple. <object> tiene fallback en su contenido interior. Usados para PDFs y contenido interactivo.

<embed src="doc.pdf" type="application/pdf" width="600" height="800"/>

// 13 · APIs Modernas

APIs de HTML5

HTML5 no solo agregó etiquetas — introdujo un conjunto de APIs JavaScript nativas que permiten funcionalidades antes imposibles sin plugins. Estas APIs abren las puertas a aplicaciones web verdaderamente poderosas.

Canvas API

Dibuja gráficos 2D con JavaScript. La base de la mayoría de juegos web y visualizaciones de datos. Context 2D para gráficos planos, WebGL para 3D acelerado por GPU.

const ctx = canvas.getContext('2d');
ctx.fillStyle = 'red';
ctx.fillRect(10, 10, 100, 80);

Geolocation API

Obtiene la ubicación geográfica del usuario (GPS o IP). Requiere permiso explícito. Fundamental para apps de mapas, delivery y servicios basados en ubicación.

navigator.geolocation.getCurrentPosition(
pos => console.log(pos.coords.latitude)
);

Web Storage

localStorage (persistente) y sessionStorage (por sesión) almacenan datos clave-valor en el navegador. Hasta 10MB. Sin fecha de expiración en localStorage.

localStorage.setItem('usuario', 'Ana');
const u = localStorage.getItem('usuario');

Drag & Drop API

Permite arrastrar y soltar elementos HTML de forma nativa. Eventos: dragstart, drag, dragend, dragover, drop. Fundamental para interfaces de tableros (Kanban).

<div draggable="true"
ondragstart="mover(event)">Arrastrar</div>

Web Workers

Ejecuta JavaScript en hilos secundarios sin bloquear la UI. Ideal para cálculos pesados, procesamiento de imágenes y operaciones que toman mucho tiempo.

const worker = new Worker('worker.js');
worker.postMessage({ datos: [1,2,3] });
worker.onmessage = e => console.log(e.data);

WebSockets

Comunicación bidireccional en tiempo real entre navegador y servidor. La base de chats, juegos multijugador, paneles en vivo y notificaciones push.

const ws = new WebSocket('wss://server.com');
ws.onmessage = e => console.log(e.data);
ws.send('Hola servidor');

Fetch API

Solicitudes HTTP modernas basadas en Promesas. Reemplaza XMLHttpRequest. Junto con async/await es la forma estándar de comunicarse con APIs REST.

const res = await fetch('/api/datos');
const json = await res.json();

Notifications API

Envía notificaciones del sistema al usuario fuera del navegador. Requiere permiso. Muy usado en PWA (Progressive Web Apps) para engagement.

Notification.requestPermission().then(() => {
new Notification('Hola!', { body: 'mensaje' });
});

// 14 · Accesibilidad

HTML Accesible

La accesibilidad web garantiza que personas con discapacidades puedan usar tus páginas. El 95% de los problemas de accesibilidad tienen solución en HTML semántico correcto. Es también un requisito legal en muchos países.

→

Usar <button> en lugar de <div onclick> es accesible por defecto: es focusable con teclado, activable con Enter y Space, y anunciado correctamente por lectores de pantalla.

→

Toda imagen debe tener el atributo alt. Si la imagen es decorativa, usa alt="". Si contiene información, describe esa información. Los lectores de pantalla leen el alt en lugar de la imagen.

→

El contraste de color entre texto y fondo debe ser al menos 4.5:1 para texto normal y 3:1 para texto grande, según WCAG 2.1 nivel AA.

→

Usa aria-label para elementos sin texto visible: botones con solo iconos, inputs sin label visible, widgets interactivos complejos.

→

La jerarquía de encabezados h1→h2→h3 no debe saltarse niveles. Los usuarios de lectores de pantalla navegan por encabezados para orientarse en la página.

// 15 · Profesional

Buenas Prácticas HTML5

Semántica sobre divs

Usa la etiqueta correcta para cada tipo de contenido. Un artículo es <article>, no un <div class="articulo">. La semántica mejora SEO, accesibilidad y mantenibilidad.

Indentación consistente

Indenta cada nivel de anidamiento con 2 o 4 espacios. El código legible es código mantenible. Usa editores como VS Code con formato automático.

Atributos en minúsculas

Aunque HTML no distingue mayúsculas, es estándar escribir etiquetas y atributos en minúsculas: <div class="..."> no <DIV CLASS="...">.

Alt en todas las imágenes

Siempre incluir alt en <img>. Descriptivo si la imagen tiene significado. Vacío (alt="") si es puramente decorativa. Nunca omitir el atributo.

Validar el código

Usa el validador W3C en validator.w3.org. HTML inválido puede renderizarse diferente en distintos navegadores. La validación previene bugs difíciles de diagnosticar.

Un solo H1 por página

El <h1> define el tema principal. Solo debe haber uno. Los motores de búsqueda usan el H1 como señal principal de relevancia del contenido.

CSS en head, JS al final

Los estilos van en el <head> para evitar FOUC (Flash of Unstyled Content). Los scripts van antes de </body> o con atributo defer para no bloquear la carga.

Separar estructura, estilo y comportamiento

HTML para estructura, CSS para presentación, JS para comportamiento. Evita inline styles y onclick en HTML. Esta separación de responsabilidades facilita el mantenimiento.

Optimizar imágenes

Usa formatos modernos (WebP), declara dimensiones en el HTML para evitar Layout Shift, y usa el atributo loading="lazy" para carga diferida de imágenes fuera de pantalla.

<img loading="lazy" width="800" height="600" alt="..."/>

Comentarios útiles

Usa comentarios para marcar el cierre de secciones largas (<!-- /header -->) o para explicar código complejo. No comentes lo obvio.

<!-- Inicio: sección hero -->
<section class="hero">...
</section><!-- /hero -->

Herramientas recomendadas para HTML: Editor VS Code con extensión Live Server · Validador W3C (validator.w3.org) · Chrome DevTools (F12) para inspeccionar el DOM · Can I Use (caniuse.com) para compatibilidad de navegadores · Lighthouse para auditoría de rendimiento y accesibilidad.

"El HTML bien escrito es invisible — funciona tan bien que nadie lo nota. El HTML mal escrito es el primero en fallar."
